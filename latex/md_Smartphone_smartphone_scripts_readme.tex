This project handles incoming I\+MU sensor data from an Android smartphone and performs data clipping and gravity influence removal. It then publishes on the {\itshape smartphone} topic the filtered result.\hypertarget{md_Smartphone_smartphone_scripts_readme_autotoc_md29}{}\doxysection{How to install the package}\label{md_Smartphone_smartphone_scripts_readme_autotoc_md29}

\begin{DoxyEnumerate}
\item Unzip {\itshape org.\+ros.\+android.\+android\+\_\+tutorial\+\_\+camera\+\_\+imu\+\_\+1.\+0.\+apk} in order to install the {\itshape Camera\+Imu} app in your smartphone. Warning\+: the app works best with Android 8.\+1 or older; earlier OS versions may cause frequent freezes/crashes
\item Install the following libraries\+: {\itshape numpy}, {\itshape matplotlib}, {\itshape pandas} (you can use {\bfseries{pip list}} in terminal, beforehand, to check if these are already installed)\+:
\begin{DoxyEnumerate}
\item Type {\bfseries{python -\/m pip install numpy}} in terminal to install numpy. Check \href{https://scipy.org/install.html}{\texttt{ Sci\+Py.\+org}} for more information.
\item Type {\bfseries{python -\/m pip install -\/U matplotlib}} in terminal to install matplotlib. Head over to \href{https://matplotlib.org/3.3.0/users/installing.html}{\texttt{ matplotlib.\+org}} for further information.
\item Type {\bfseries{sudo apt-\/get install python-\/pandas}}. In case of errors, you can take a look \href{https://pandas.pydata.org/pandas-docs/version/0.13.1/install.html}{\texttt{ here}} for more information.
\end{DoxyEnumerate}

\begin{DoxyEnumerate}
\item Launch R\+OS by typing {\bfseries{roscore}} in a terminal.
\item Start the {\itshape Camera\+Imu} app in your smartphone. Then you have to type the IP address of your computer into the app\textquotesingle{}s {\itshape Master U\+RI} textbox (i.\+e. \href{http://YOUR_IP_ADDRESS:11311}{\texttt{ http\+://\+Y\+O\+U\+R\+\_\+\+I\+P\+\_\+\+A\+D\+D\+R\+E\+S\+S\+:11311}}, as the default port is {\itshape 11311}). In order to find your IP address, type {\bfseries{hostname -\/I}} in terminal if you are using a Unix system, or type {\bfseries{ipconfig}} in command prompt if using Windows.
\begin{DoxyItemize}
\item It is possible to open another terminal and subscribe to the {\itshape android/imu} topic, by typing {\bfseries{rostopic echo /android/imu}}, in order to check the incoming data.
\end{DoxyItemize}
\item In a second terminal, type {\bfseries{rosrun \mbox{\hyperlink{namespaceimu__calib}{imu\+\_\+calib}} do\+\_\+calib}} and start calibrating the smartphone I\+MU, following on-\/screen instructions. Then, apply the performed calibration by using {\bfseries{rosrun \mbox{\hyperlink{namespaceimu__calib}{imu\+\_\+calib}} apply\+\_\+calib}}. The calibration parameters are stored in {\itshape imu\+\_\+calib.\+yaml}, available in the current user\textquotesingle{}s home folder.
\begin{DoxyItemize}
\item Again, it is possible to open another terminal and type {\bfseries{rostopic echo /android/imu\+\_\+corrected}}, in order to view the incoming calibrated data.
\end{DoxyItemize}
\item In a third terminal, type {\bfseries{rosrun smartphone compute\+Gravity.\+py}} to start collecting I\+MU data and removing the gravity influence from incoming linear acceleration samples. Notice that this step is needed only if executing the whole {\itshape Sof\+Ar-\/project}. Otherwise, gravity gets already neglected in the next step.
\item In a fourth terminal, launch {\bfseries{rosrun smartphone clipping.\+py}} to start clipping I\+MU data, when needed, and removing the gravity influence from incoming linear acceleration samples.
\begin{DoxyItemize}
\item Remember to set {\itshape flag\+Write\+Data = 1} in the python file if you want to analyze incoming data through plots. Otherwise set {\itshape flag\+Write\+Data = 0} to skip the offline analysis. If {\itshape flag\+Write\+Data} is set to 1, then also run {\bfseries{rosrun smartphone offline\+Analysis.\+py}} in yet another terminal, in order to plot the data obtained by running {\itshape clipping.\+py} 
\end{DoxyItemize}
\end{DoxyEnumerate}
